`deeptutor/learning/prompts.py`

## 要求

```python
"""Guided Learning prompt templates — Framework v1.8.2."""

DIAGNOSTIC_PHASE1_SYSTEM = """你是一个教育诊断专家...
返回 JSON 格式：{"questions": ["Q1", "Q2"], "answers": ["A1", "A2"]}"""

DIAGNOSTIC_PHASE1_USER = "生成摸底测试题，覆盖基础概念"

EXPLAIN_SYSTEM = """你是一个耐心专业的老师。请讲解以下知识点。
要求：1. 生活化比喻引入 2. 核心概念定义 3. 1-2个例子 4. 常见误解 5. 300-500字"""

EXPLAIN_USER = "讲解知识点：{knowledge_point}"

PRETEST_SYSTEM = """请出一道预习题。
返回 JSON：{"question": "...", "hint": "..."}"""

PRETEST_USER = "为知识点出预习题：{knowledge_point}"

FEYNMAN_SYSTEM = """判断学生是否能用费曼技巧解释清楚概念。
返回 JSON：{"passed": true/false, "feedback": "...", "gap": "..."}"""

FEYNMAN_USER = "检验对概念的费曼解释：{knowledge_point}"

PRACTICE_SYSTEM = """生成3-5道练习题，难度递进（识记→理解→应用→分析）。
返回 JSON：{"exercises": [{"question": "...", "answer": "...", "bloom_level": "..."}]}"""

PRACTICE_USER = "为模块生成练习题：{module_name}"

ERROR_DIAGNOSIS_SYSTEM = """分析学生做错的题目属于什么错误类型。
返回 JSON：{"error_type": "知识结构性/理解偏差型/应用错误/元认知型", "analysis": "...", "remediation": "..."}"""

ERROR_DIAGNOSIS_USER = "分析这道题的答题错误"

MODULE_TEST_SYSTEM = """出一套10题模块测验，覆盖所有知识点，通过线70%。
返回 JSON 格式。"""

MODULE_TEST_USER = "为模块生成测验：{module_name}"

REVIEW_SYSTEM = """生成间隔复习内容：核心概念回顾、易错点提醒、综合练习。"""

REVIEW_USER = "生成复习内容"

METACOGNITIVE_SYSTEM = """介绍高效学习方法：主动回忆、间隔重复、费曼技巧。"""

METACOGNITIVE_USER = "介绍学习方法"

PLAN_SYSTEM = """基于诊断结果制定学习计划，按优先级排列模块学习顺序。"""

PLAN_USER = "制定学习计划"
```

可以调整 prompt 质量（让它们更详细），但保持以上结构（每个 stage 有 SYSTEM 和 USER 两个常量）。

## 修改文件

`deeptutor/capabilities/guided_learning.py` — 替换临时 prompt 字符串为从 prompts.py 导入的常量：

```python
from deeptutor.learning.prompts import (
    DIAGNOSTIC_PHASE1_SYSTEM, DIAGNOSTIC_PHASE1_USER,
    EXPLAIN_SYSTEM, EXPLAIN_USER,
    PRETEST_SYSTEM, PRETEST_USER,
    ...等
)
```

每个 handler 中用对应常量替代内联字符串。例如：
```python
# Before (Block 2)
response = await self._call_llm(
    '判断学生是否能用费曼技巧解释清楚概念。返回JSON{"passed":bool,"feedback":"...","gap":"..."}',
    "费曼检验",
)

# After (Block 3)
response = await self._call_llm(FEYNMAN_SYSTEM, FEYNMAN_USER)
```

## 注意

- 不要再改 `_call_llm` 或任何其他方法，只替换 prompt 字符串
- 保持 stage handler 结构不变
